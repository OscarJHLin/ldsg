# 分层动态空间图记忆系统 v2.0

版本 2.0 | 2026年4月

> **设计原则：一图一宇宙，无分层之名，有分化之实。**

---

## 摘要

传统记忆系统将"短期记忆"与"长期记忆"视为两个独立存储区，通过定期归档实现知识沉淀。
LDSG v2.0 将这一范式彻底颠覆：**记忆只有一张大图，区分只在于激活程度与连接密度。**

新架构的核心突破：
- **子图隔离**：每个任务产生独立子图，初始与主图物理隔离
- **投影空间作为工作台**：多子图并行映射，按需切换
- **连接即学习**：相似性发现 → 原图上真正建立连接
- **削减仅在投影空间**：原图只增不减，削减只控制投影空间规模

---

## 核心概念

### 1. 原图（主图）

> 长期记忆的本体，唯一真理源。

- 存储所有节点、所有连接
- **只增不减**（除非遗忘规则触发）
- 削减操作**不在原图进行**
- 连接建立后永久存在，除非整体遗忘

### 2. 子图（任务子图）

> 每个任务产生的记忆块，初始与主图隔离。

- 来自一次对话、一个任务、一段工作流
- 初始状态：**物理隔离**，不与主图相连
- 结构：`{ nodes: [], edges: [], task_id, created_at, weight }`
- 包含该任务中学到的所有节点和关联

### 3. 投影空间（工作台）

> 正在工作中的子图集合，最多同时 3 个。

- 性质：**临时工作区**，不是持久存储
- 规则：**最多 3 个子图同时在**
- 新子图进入 → 从权重最低的子图被挤出
- 挤出不删除子图，只是移出投影空间

### 4. 索引表（短期记忆目录）

> 类似内存结构，快速定位投影空间里的子图。

- 结构：`{ subgraph_id → { position, summary, node_count, weight } }`
- **不是图结构**，是简单的kv目录
- 作用：快速知道"哪个子图在投影空间里，它大概是什么内容"

### 5. 投影

> 子图进入投影空间的过程。

- 子图被加载到投影空间
- 与其他子图、主图节点共同参与相似度计算
- 投影后可以进行锚定、遍历等操作

---

## 核心操作

### A. 遍历与锚定

**工作时，两条线同时跑：**

```
1. 从索引表出发 → 找到相关子图 → 锚定 → 展开子图节点

2. 主图也在遍历 → 已激活的节点 → 与子图节点对比 → 寻找相似
```

**空闲时（随机游走）：**
- 在原图上随机游走
- 发现子图与原图有相似节点 → 在**原图**上建立连接

### B. 连接建立

> 什么时候、在哪里、怎样建立连接？

**触发条件：**
- 工作遍历中发现子图节点与主图节点相似（向量相似度 > 阈值）
- 空闲随机游走中发现跨子图的连接机会
- 多轮任务复用同一子图时强化连接

**建立位置：**
- **在原图上真正建立连接**
- 不是投影空间里的临时映射
- 连接一旦建立，永久存在

**连接过程：**
```
子图节点 A ---相似度--> 主图节点 B
        ↓
在原图上建立: A ↔ B 双向连接
        ↓
子图 weight += 1（与主图融合程度提升）
```

### C. 子图融入

> 什么时候子图可以融入主图？

**融入条件：**
- 子图在投影空间中被频繁锚定（≥3次）
- 子图与主图建立了多个有效连接（≥2个）
- 空闲时间可用

**融入方式：**
- 子图从"孤立子图"状态解除
- 与主图有连接的节点并入主图
- 子图内部连接保持完整
- 原图 weight 重新归一化

### D. 削减（投影空间）

> 削减的是**投影空间**，不是原图。

**触发：** 投影空间超过 3 个子图

**策略：** 从权重最低的开始挤出

```
投影空间: [子图A(0.9), 子图B(0.7), 子图C(0.6), 子图D(0.4)]
                            ↓
                        挤出自图D
                            ↓
投影空间: [子图A(0.9), 子图B(0.7), 子图C(0.6)]
```

**削减后：**
- 子图D 回到"孤立子图"状态
- 索引表中移除 D 的条目
- 原图**完全不动**

### E. 遗忘（原图）

> 削减的是**原图**，在遗忘规则触发时。

**触发条件：**
- 节点长期未激活（超过活跃阈值）
- 节点权重低于遗忘阈值
- 与其他节点无连接（孤立节点）

**策略：**
- 按权重从低到高剪除
- 先剪孤立节点，再剪弱连接节点
- 重要节点（高权重、高连接度）永不主动删除

**遗忘后：**
- 节点从原图移除
- 相关连接一并清除
- 触发主图结构的归纳整理

---

## 三个空间的关系

| 空间 | 作用 | 容量 | 削减规则 |
|------|------|------|---------|
| **原图** | 长期记忆本体 | 无上限 | 遗忘规则触发时 |
| **投影空间** | 工作台，多子图并行 | ≤3 个子图 | 超量时挤出 |
| **索引表** | 快速定位子图 | 与投影空间联动 | 无需削减 |

## 索引结构

### 节点结构

```python
Node {
    id: str
    concept: str              # 节点核心语义
    vector: List[float]        # 向量表示
    weight: float              # 激活权重 0.0~1.0
    connections: List[str]       # 连接的节点ID
    subgraph_id: str | None     # 所属子图（None=主图）
    created_at: float          # 创建时间戳
    last_activated: float     # 最后激活时间
}
```

### 子图结构

```python
Subgraph {
    id: str
    task_id: str              # 来源任务标识
    nodes: List[str]          # 节点ID列表
    weight: float             # 权重（与主图融合程度）
    isolated: bool            # 是否与主图隔离
    created_at: float
}
```

### 投影空间结构

```python
ProjectionSpace {
    subgraphs: List[str]      # 当前子图ID列表（最多3个）
    index_table: Dict[str, SubgraphIndex]  # 索引表
}
```

## 遗忘与连接的生命周期

```
节点生命周期：

创建 → 激活 → 使用 → 休眠 → 遗忘
   ↑                      ↓
   └── 反复激活 ──────────┘
       (权重上升)

连接生命周期：

建立 → 强化 → 稳定 → 衰减 → 断裂
   ↑                              ↓
   └── 反复使用 ───────────────────┘
       (连接强度上升)
```

## 与 v1.0 的核心差异

| 维度 | v1.0 | v2.0 |
|------|-------|-------|
| 图结构 | 三层分离（STM/MSG/WS） | 单一主图 + 投影空间 |
| 削减位置 | 三锚点压缩栈 | 投影空间（≤3子图） |
| 原图变化 | 频繁更新 | 只增不减 |
| 连接建立 | 预设规则 | 发现式建立 |
| 子图状态 | 不隔离 | 初始隔离，融入时解除 |
| 遗忘 | 动态压缩栈 | 原图遗忘规则 |
